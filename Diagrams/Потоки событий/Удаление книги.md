# Поток событий для прецедента "Удаление(списание) книги":



### Основной поток:

1. Библиотекарь пытается войти в систему. 
2. Вход успешно. 
3. Библиотекарь пытается выполнить поиск записи для определенной книги. 
4. Запись найдена. 
5. Он / она хочет удалить запись. 
6. Запись была удалена.

### Альтернативный поток событий:
1. Если логин введен неверно Библиотекарь должен  повторить попытку. 
2. Если поиск книги не увенчался успехом, то эту книгу нельзя просмотреть и удалить.
